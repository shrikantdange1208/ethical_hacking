What is Active Directoy?

Developed by Microsoft. It is like a phone book. Stores information.
You can have 1 username and password. You use same username and password to access different applications.
Authentication is done using Kerberos tickets.
Non-windows devices, such as Linux machines, firewalls, etc. can also authenticate to Active Directory using RADIUS or LDAP


Why Active Directory?
1. Common Identity Management Service
2. Can be exploited without ever attacking patchable exploits.
    Instead, we abuse features, trusts, components and more.

Physical Active Directory Components:

Domain Controller:
  Host a copy of AD Domain Services (DS) directory store.
  Provides authentication and authorizaton.
  Replicate updates to other domain controllers in the domain and forest
  Allow admin access to manage user accounts and network resources.
 
The AD DS data store:
  Consists of Ntds.dit file
  Default location %SystemRoot%\NTDS folder on all domain server (Contains all uses, objects, groups,etc. Also contains all password hashes in that domain)
  Is accessible only through the domain controller processes and protocols

Logical Active Directory Components:

AD DS Schema
  Rule book or a blue print.
  Defines every object that can be created in AD.
  Enforces rules regarding object creation and configuration

Domains
  Used to group and manage objects in an org.
  Administrative boundary for applying policies to groups of objects
  Replication boundary for replicating data between domain controllers
  Authentication and Authorization boundary that provides a way to limit the scope of access to resources.

Trees
  Group of domains
  Domain tree is heirarchy of domains in AD DS.
  Share a contiguous namespace with parent domain
  Can have addiitional child domains
  Creates a two-way transitive trust with other domains

Forests
  Collection of trees.
  Share a common schema.
  Share a common configuration
  Share some global catalog to enable searching
  Enable trusts between all domains in the forest
  Share the Enterprise Admins and Schema Admin groups
 
Organizational Units (OU) 
  OU and AD containers that can contain users, groups,computers and other OUs.
   Represents organization hierarchically and logically
   Delegate permissions to administer groups of objects
   Apply policies

Trusts

  Provides mechanism for users to gain access to resources in another domain  
  Directional Trusts - Follows from trusting domain to trusted domain
  Transitive Trusts - Relationship is extended beyond a two-domain trust to include other trusted domains.

All domains in forest trust all other domains in the forest
Trusts ca extend outside the forest
